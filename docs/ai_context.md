# AI Context for LearnWithNura

## Purpose
This document is the primary AI onboarding document for the LearnWithNura repository.
It will provide a single source of truth for the project mission, core philosophy, educational pipeline, and AI-specific instructions.

## Project Mission

**Primary Mission**: Transform structural complexity into hyper-simplified, high-relevance educational frameworks designed for distribution in the digital creator economy.

**Secondary Mission**: Build the LearnWithNura Visual Education Operating System—a comprehensive platform that takes technical concepts (AI, Web3, Crypto) and breaks them down into visually intelligent content explainable to 10-year-old learners through trusted authority positioning.

**Creator Economy Focus**: Provide tools and systems enabling individual creators to establish themselves as authoritative educators through consistent, jargon-free visual education delivered across multiple platforms (PowerPoint, social media carousels, written threads, video scripts, guides).

**Repository Support**:
- Generate professional PowerPoint presentations from structured content
- Apply educational simplification across all content
- Support multi-format output (beyond PowerPoint)
- Enable rapid content iteration and testing
- Provide metrics and feedback for optimization
- Manage localized, culturally-relevant content

## Core Philosophy

**Educational Philosophy: The Simplification Mandate**

All content passes through a 5-tier filter:
1. **Simple Child-Like Analogy**: Zero prerequisite knowledge
2. **Hausa Language Anchor**: Deep localization for psychological safety
3. **Low-Fidelity Visual Modeling**: ASCII/text-based structural visualization
4. **High-Fidelity AI Prompt Templates**: Automated generation systems
5. **Verification Checklist**: Strict structural confirmation and jargon-free validation

**Core Principle**: "If you cannot sketch it simply on an index card, you do not understand it systematically."

**Modularity & Extensibility**:
- Plugin-based architecture for layouts, themes, and providers
- New educational engines added without modifying core
- Reusable components across all layouts
- Template system for rapid content creation

**Accessibility & Clarity**:
- Zero unexplained technical terminology
- Cognitive overload prevention through atomic unit presentation
- Multiple learning modes (visual, textual, interactive)
- Cultural and linguistic adaptation built-in

**Backward Compatibility First**:
- All existing slides.txt files remain functional
- New features extend rather than replace
- Breaking changes only after deprecation period

## Educational Pipeline

**End-to-End Workflow**:

1. **Content Planning**
   - Identify learning concept
   - Research technical mechanisms
   - Gather real-world examples for analogies
   - Plan cognitive progression

2. **Content Generation (Multi-Engine)**
   - **Simplification**: Break into child-like components
   - **Analogy Creation**: Generate 3 distinct real-world mappings
   - **Localization**: Hausa translation and cultural anchoring
   - **Visual Modeling**: ASCII flows and concept cards
   - **Prompt Engineering**: RCC framework for AI enhancement

3. **Educational Verification**
   - Jargon detection and elimination
   - Complexity scoring (must pass 10-year-old rule)
   - Risk mapping and disclosure
   - Knowledge verification checklist generation

4. **Multi-Format Production**
   - PowerPoint deck generation (primary)
   - 6-Slide carousel creation
   - 5-Part social thread scripting
   - Video outline generation
   - Guide/PDF compilation

5. **Content Optimization**
   - Algorithm psychology application (bookmarks, dwell time)
   - Hook implementation (counter-intuitive, process reveal, anti-complexity)
   - Authority positioning and brand voice
   - Platform-specific adaptation

6. **Distribution & Feedback**
   - Simultaneous release across platforms
   - Metric collection (engagement, saves, comments)
   - Audience confusion point identification
   - Iterative improvement based on feedback

## Design Principles

**Architectural Principles**:
1. **Plugin-First**: All extensions through plugins, zero core modifications
2. **Blueprint-Aligned**: Every feature serves a blueprint requirement
3. **Educational-First**: Learning effectiveness over implementation ease
4. **Data-Driven**: All decisions informed by engagement metrics
5. **Extensible**: New modules don't break existing systems

**Code Quality Standards**:
- All code must support backward compatibility
- Documentation before features
- Reusable components over one-off implementations
- Simplicity over complexity
- Testing coverage before deployment

**Component Design**:
- Clear interface contracts between modules
- Auto-discovery registry patterns
- Dependency injection for flexibility
- Separation of concerns (rendering, content, strategy)

**API Philosophy**:
- CLI-first for discoverability
- Programmatic API for automation
- Configuration over convention
- Sensible defaults with override capability

## AI Instructions

**How AI Agents Should Approach Development**:

1. **Blueprint as Authority**
   - Visual Educator Blueprint v3.0 is the authoritative guide for all future development
   - Use `docs/blueprint_extraction_summary.md` to understand blueprint concepts
   - All features should be justified against blueprint requirements

2. **Educational Verification Before Implementation**
   - Use `docs/gap_analysis_comprehensive.md` to identify priority gaps
   - Follow `docs/future_architecture_mapping.md` for system design
   - Never implement features that bypass verification systems

3. **Documentation-First Approach**
   - Create comprehensive documentation before coding
   - Use provided AI prompt templates (`prompts/master_prompt.md`)
   - Document why decisions are made, not just what was done

4. **Backward Compatibility Mandatory**
   - All changes must preserve existing slides.txt compatibility
   - Deprecate before removing
   - Test against historical content

5. **Testing & Verification**
   - Add tests for every new module
   - Verify jargon-free output at 100%
   - Track complexity scores and cognitive load metrics

6. **Integration Points**
   - Extend providers for new content generation modes
   - Add to layout registry for new visual templates
   - Use theme system for brand/design consistency

7. **Code Quality Standards**
   - Follow repository conventions (see `development_rules.md`)
   - Modular design with clear interfaces
   - Auto-discoverable plugins without configuration

8. **Working with This Repository**:
   - Read `docs/current_system_analysis.md` to understand existing architecture
   - Use `docs/roadmap.md` to align with transformation phases
   - Reference `docs/educational_philosophy.md` for content principles
   - Consult `development_rules.md` for engineering standards
